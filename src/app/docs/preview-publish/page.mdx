# Preview & Publish

Once you've built your portfolio in the Studio Editor, preview it to check how it looks and publish it to make it live for the world to see.

## Preview Mode

### From the Editor

1. Click the **Preview** button (eye icon) in the editor's top bar
2. A full-screen preview opens showing your portfolio exactly as visitors will see it
3. All blocks are rendered with their final styles, animations, and layouts
4. Interactive elements work — click buttons to test scroll-to-section links, external links open in new tabs, contact forms are functional
5. Press **Escape** or click the **close button** to return to the editor

### Device Preview

Before going full-screen, use the **device toggles** in the editor's top bar to simulate different screen sizes:

- **Desktop** (monitor icon) — full-width canvas, typically 1440px
- **Tablet** (tablet icon) — medium width, simulates iPad-size screens
- **Mobile** (phone icon) — narrow width, simulates phone screens

This helps you check:
- Whether your layout collapses properly on small screens
- If blocks set to **Hide on Mobile** or **Hide on Desktop** behave correctly
- Whether text is readable at smaller sizes
- If touch targets (buttons, links) are large enough for finger taps

### From the Dashboard

You can also preview from the **Portfolios** page without opening the editor:
1. Go to **Portfolios** in the sidebar
2. Click the **eye icon** on any portfolio card
3. The preview opens in a new view

## Publishing

Publishing makes your portfolio accessible to anyone with the URL. Only published portfolios are visible to the public.

### First-Time Publish

1. Open your portfolio in the Studio Editor
2. Click the **Publish** button in the top-right corner of the top bar
3. Foliocraft automatically:
   - Saves all pending changes
   - Creates a version snapshot labeled "Published" (so you can always revert)
   - Sets the portfolio status to **Published**
4. Your portfolio is now live at its URL

### Updating a Published Portfolio

After the first publish, the button changes to **Update & Publish**:

1. Make your changes in the editor
2. Click **Update & Publish**
3. All changes go live immediately
4. A new version snapshot is created automatically

### Draft Mode

If you're not ready to publish:
- Your portfolio stays in **Draft** status
- Only you can see it (it's not accessible via the public URL)
- You can keep editing at your own pace
- Publish whenever you're ready

> **Tip:** You can unpublish a portfolio by changing its status back to Draft. This removes public access but keeps all your content.

## Portfolio URLs

### URL Structure

Your published portfolio is available at:

`yourdomain.com/portfolio/your-username/portfolio-slug`

For example: `foliocraft.dev/portfolio/alexchen/main`

### Default Portfolio URL

If you set a default portfolio in **Settings > Preferences > Default Portfolio**, it's also accessible at:

`yourdomain.com/portfolio/your-username`

This shorter URL is ideal for sharing on social media, resumes, and business cards.

### Sharing Your Portfolio

After publishing, share your portfolio by:
1. Clicking the **Copy URL** button on the portfolio card in the dashboard
2. Using the **external link icon** to open it in a new tab and copying the URL from the browser
3. Adding it to your social profiles, resume, email signature, or LinkedIn

## Exporting

Export your portfolio data from the editor's **File menu**:

### Export as JSON
Downloads a complete JSON file containing:
- All sections with their layout settings
- All blocks with content and styles
- Theme settings
- Metadata (title, description, slug)

The JSON export is useful for:
- Backing up your portfolio
- Transferring content to another account
- Programmatic access to your portfolio data

### Export as HTML
Downloads a standalone HTML file that can be:
- Opened directly in a browser
- Hosted on any static file server
- Shared as a single file

## Publishing Checklist

Before publishing, verify these items:

1. **Content** — all placeholder text replaced with your real content
2. **Images** — all images uploaded and displaying correctly
3. **Links** — all buttons and links point to correct URLs
4. **Section links** — scroll-to-section buttons navigate to the right sections
5. **Contact form** — test by submitting a form entry and checking the Submissions page
6. **Mobile** — use device preview to check responsive layout
7. **SEO** — set a title, description, and OG image (see [SEO & Analytics](/docs/seo-analytics))
8. **Theme** — colors, fonts, and spacing look consistent
